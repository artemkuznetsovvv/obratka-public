# Задача 07: Шаг 2.1 — Кластеризация свободных тем

## Цель

Свести разнообразные «свободные» темы (которые модель в Шаге 2 выделила вне базового набора) к единым кластерам. Один LLM-вызов независимо от объёма данных.

> v2 / tasks/12: `cluster_topics` принимает `collector: ArtifactCollector | None`,
> оборачивается в `step_span("step21")` и пишет `Step21Artifact`.

- **GPT-4o-mini** через OpenRouter

## Когда вызывается

После Шага 2 (есть результаты) и до Шага 2.2 (чтобы передать кластеризованные темы как `available_topics`).

## Системный промпт

```
Тебе дан список тем, которые модель выделила в отзывах о бизнесе.
Многие из них — синонимы или вариации одного и того же.

Сгруппируй их в кластеры. Для каждой исходной темы укажи каноническое имя.
Старайся переиспользовать уже существующие имена базового набора, если они подходят.

Базовый набор:
- еда/напитки, персонал, скорость обслуживания, чистота,
  цена/качество, атмосфера, локация/парковка, запись/коммуникация

Пример:
вход: ["стоянка", "нет места припарковаться", "wi-fi", "интернет в кафе", "бесплатный вайфай"]
выход: {
  "mapping": {
    "стоянка": "локация/парковка",
    "нет места припарковаться": "локация/парковка",
    "wi-fi": "wi-fi",
    "интернет в кафе": "wi-fi",
    "бесплатный вайфай": "wi-fi"
  },
  "canonical_topics": ["локация/парковка", "wi-fi"]
}

Возвращай строго JSON по схеме.
```

## Логика

1. Собираем уникальные `topic` из всех `aspects` Шага 2 и initial_aspects low_conf_queue, у которых `is_freeform=True` ИЛИ их нет в `BASE_TOPICS`.
2. Если уникальных «свободных» тем 0 — пропускаем шаг.
3. Один LLM-вызов с этим списком.
4. Применяем mapping к результатам Шага 2.
5. Возвращаем `available_topics = base_topics + canonical_topics` для использования в Шаге 2.2.

## Детерминированная страховка

Перед и после LLM-кластеризации применяется `obratka.utils.topics.normalize_topic`.
Она мапит очевидные синонимы в базовый набор без LLM:
- `кофе/напитки`, `кофе и напитки` → `еда/напитки`;
- служебные ошибки модели вроде `is_freeform`, `is_freeform=true` не могут
  попасть в финальный отчёт как тема и перекидываются в базовую тему по fragment.

Важно: сильная LLM из Шага 2.2 не прогоняет все свободные темы, она работает
только с low-confidence отзывами. Поэтому нормализация и Step 2.1 остаются
основным механизмом схлопывания похожих тем для high-confidence результатов.

## Критерии готовности

- [ ] При пустом списке свободных тем — шаг возвращает пустой mapping без LLM-вызова.
- [ ] `apply_topic_map` идемпотентен (повторное применение не меняет данные).
- [ ] Юнит-тест: 5 синонимов парковки → один кластер.
- [ ] Юнит-тест: тема, которая уже из базового набора, остаётся неизменной.
