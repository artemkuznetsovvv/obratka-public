## Способы экономии

1. **Batch API OpenAI** — 50% скидка на Шаги 1, 2 (но не 2.2 при стриме). Подходит, если можно ждать до 24 часов.
2. **Prompt Caching** — статические промпты в начале, переменные данные в конце. OpenRouter может не пробрасывать → проверить.
3. **Кэш по `text_hash + prompt_version`** — при ежедневном мониторинге 90–95% отзывов уже обработаны.
4. **Снизить low_conf_threshold** с 0.5 до 0.4 — меньше отзывов уйдёт в Шаг 2.2 (но качество анализа упадёт). Балансировать на проде.
5. **Few-shot в Шаге 2** — повышает качество mini-модели, снижает долю low-conf, экономит на Шаге 2.2.
