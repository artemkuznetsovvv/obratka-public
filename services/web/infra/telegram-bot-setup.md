# Telegram-бот уведомлений — создание и подключение (OBR-39)

Пошаговая инструкция: как создать бота в самом Telegram и подключить его к «Обратке».
Бот — единый канал уведомлений для двух адресатов:
- **пользователю** — алерты по его мониторингам (новые отзывы, рост негатива, частичное обновление);
- **администратору** — алерты об ошибках сбора/анализа/генерации отчёта.

Архитектура: Web API общается с Telegram по **long-polling** (фоновый сервис опрашивает `getUpdates`) —
публичный webhook-эндпоинт не нужен. Подробности решений — в [ADR-012](../../ADR-obratka-app/ADR%20по%20компонентам%20системы%20+%20описание%20системы/details/ADR-012-telegram-notifications.md).

---

## 1. Создать бота в Telegram (через @BotFather)

1. Откройте в Telegram диалог с **[@BotFather](https://t.me/BotFather)**.
2. Отправьте `/newbot`.
3. Введите **отображаемое имя** бота (любое), напр. `Обратка Уведомления`.
4. Введите **username** — он обязан заканчиваться на `bot`, напр. `obratka_alerts_bot`.
   Username должен быть свободен; именно он попадает в ссылку `https://t.me/<username>`.
5. BotFather пришлёт **token** вида `123456789:AAH...` — это секрет, не публикуйте его.

### (опционально) Оформление и команды
- `/setdescription` — описание бота.
- `/setabouttext` — короткий «about».
- `/setuserpic` — аватар.
- `/setcommands` — список команд (показывается в меню бота):
  ```
  start - привязать аккаунт
  unlink - отвязать аккаунт
  ```
- **Privacy** можно оставить включённым (`/setprivacy` = Enable) — бот общается только в личке,
  групповые сообщения ему не нужны.

---

## 2. Узнать chat_id администратора

Системные алерты шлются на жёстко заданные в конфиге `chat_id` (ручная подписка администратору не нужна).

1. Напишите своему боту любое сообщение (например, `/start`).
2. Откройте в браузере: `https://api.telegram.org/bot<TOKEN>/getUpdates`
   (подставьте токен; на проде, где `api.telegram.org` заблокирован — выполните через прокси/VPN, см. §4).
3. В ответе найдите `"chat":{"id": ...}` — это и есть ваш `chat_id` (число, может быть отрицательным).
   Альтернатива: написать боту **[@userinfobot](https://t.me/userinfobot)** — он покажет ваш id.

## 3. Прописать конфигурацию Web API

Секция `Telegram` в конфиге Web API (`appsettings.json` / `appsettings.Local.json` / переменные окружения).
Для прода значения кладите в **секреты/переменные окружения**, не в git.

```jsonc
"Telegram": {
  "Enabled": true,                       // включает канал; false → лог-стаб без отправки
  "BotToken": "123456789:AAH...",        // токен от BotFather (СЕКРЕТ)
  "BotUsername": "obratka_alerts_bot",   // username без @ — для deep-link привязки
  "AdminChatIds": [ "123456789" ],       // chat_id администратора(ов), строками
  "DashboardBaseUrl": "https://obratka.example.com",  // базовый URL фронта для кнопки «Открыть дашборд»
  "Proxy": ""                            // прокси для api.telegram.org (см. §4); пусто = прямой доступ
}
```

Через переменные окружения (docker/прод) те же значения:
```
Telegram__Enabled=true
Telegram__BotToken=123456789:AAH...
Telegram__BotUsername=obratka_alerts_bot
Telegram__AdminChatIds__0=123456789
Telegram__DashboardBaseUrl=https://obratka.example.com
Telegram__Proxy=socks5://proxy-host:1080
```

> **Поведение без конфига:** если `Enabled=false` или `BotToken` пуст — модуль уведомлений работает
> «лог-стабом»: ничего не отправляет, только пишет в Serilog `[telegram:send] ... skipped`. Так dev-стенд
> поднимается без бота. По умолчанию в `appsettings.json` канал **выключен**.

---

## 4. ВАЖНО: доступ к `api.telegram.org` на хостинге

На проде `api.telegram.org` **заблокирован хостингом** — прямой запрос не пройдёт. Нужен один из путей:

- **Прокси (рекомендуется для приложения).** Укажите `Telegram:Proxy` — `socks5://host:port` или
  `http://host:port`. Web API соберёт единый `HttpClient` с этим прокси; **через него идут И отправка
  сообщений (`sendMessage`), И приём команд (long-poll `getUpdates`)**. Проверьте, что из контейнера
  Web API доступен сам прокси-хост.
- **VPN-route на хосте/в контейнере** (WireGuard и т.п.) — делает `api.telegram.org` достижимым прозрачно.
  Тогда `Telegram:Proxy` оставьте пустым; изменений в приложении не требуется.
- **В dev (Windows)** обычно есть прямой доступ → `Telegram:Proxy` пустой.

Зафиксируйте выбранный путь и адрес прокси/VPN в секретах окружения.

## 5. Запуск и проверка

1. Перезапустите Web API (бот-receiver стартует только при `Enabled=true` и заданном токене — в логе
   появится «Telegram long-poll receiver запущен (бот @...)»).
2. **Привязка пользователя:** личный кабинет → **Уведомления** → «Подключить Telegram» → открывается бот →
   нажмите **Start**. Бот ответит «✅ Telegram привязан», страница обновится (статус «привязан»).
3. **Уведомление о цикле:** на дашборде включённого мониторинга → «Обновить вручную» → по завершении в бота
   придёт «🔄 Обновление выполнено … +N новых отзывов» с кнопкой «📊 Открыть дашборд».
4. **Admin-алерт:** при ошибке сбора/анализа/отчёта администратору(ам) из `AdminChatIds` придёт
   «⚠️ Ошибка: …» с полями (пользователь, компания, этап, причина, статус, event id).
5. **Отвязка:** Уведомления → «Отвязать Telegram» (с подтверждением) либо команда `/unlink` боту.

## 6. Команды бота
| Команда | Действие |
|---|---|
| `/start <token>` | Привязать аккаунт (token приходит из deep-link кабинета). Без токена — подсказка. |
| `/unlink` (или `/stop`) | Отвязать текущий чат от аккаунта. |
| `/chatid` | Показать `chat_id` текущего чата — удобно, чтобы добавить группу в `Telegram:AdminChatIds`. |
| `/wisdom` или фраза «Обратка, мудрость» | Тестовая команда: бот отвечает мыслью — подтверждает, что он жив (в т.ч. в группе). |
| `/testalert` | Диагностика: из чата, что есть в `Telegram:AdminChatIds`, шлёт тестовый admin-алерт тем же путём, что и реальные ошибки. Из не-admin чата — откажет. |

Для `/setcommands` в BotFather можно задать:
```
start - привязать аккаунт
unlink - отвязать аккаунт
chatid - показать id чата
wisdom - мысль для проверки
```

## 7. Алерты в группу

Системные алерты идут на все `Telegram:AdminChatIds`. Туда можно положить **id группы** — тогда алерты
будут приходить в группу.

1. Создайте группу в Telegram и добавьте в неё своего бота (Add members → найдите бота по username).
2. Напишите в группе **`/chatid`** — бот ответит `chat_id этого чата` (у групп он отрицательный,
   у супергрупп вида `-100…`). Если бот не реагирует на команду в группе — он, возможно, не видит
   обычные сообщения из-за privacy: команды (`/chatid`, `/wisdom`) бот получает всегда, а вот **фраза
   «Обратка, мудрость» в группе сработает только при выключенном privacy** (`@BotFather` → `/setprivacy`
   → выбрать бота → **Disable**; затем удалить и заново добавить бота в группу).
3. Добавьте полученный id в `Telegram:AdminChatIds` (строкой), например:
   ```jsonc
   "AdminChatIds": [ "123456789", "-1001234567890" ]   // личка админа + группа
   ```
4. Перезапустите Web API. Теперь admin-алерты об ошибках приходят и в группу.

> Привязка **пользователя** (`/start <token>`) обычно делается в личке бота. Технически можно привязать
> и групповой чат (откройте deep-link, добавив бота в группу) — тогда user-уведомления пойдут в группу.

## 8. Локальный запуск (для проверки бота)

1. Поднять инфраструктуру (Postgres для `webapi_db` + Seq для логов):
   ```bash
   docker compose -f Web/infra/docker-compose.dev.yml up -d
   ```
   (Postgres на `localhost:5432`, Seq на `http://localhost:5341`. `processing_db` для теста бота не нужен —
   Analytics-сервисы опциональны.)
2. Вписать в `Web/backend/src/Obratka.WebApi/appsettings.Local.json` секцию `Telegram`:
   `BotToken`, `BotUsername`, `AdminChatIds` (свой chat_id и/или id группы), `DashboardBaseUrl` (`http://localhost:5173`).
3. Запустить Web API (миграции применятся автоматически при старте):
   ```bash
   dotnet run --project Web/backend/src/Obratka.WebApi
   ```
   В логе должно появиться: `Telegram long-poll receiver запущен (бот @...)`.
4. Запустить фронтенд:
   ```bash
   cd Web/frontend && npm install && npm run dev
   ```
   Открыть `http://localhost:5173`, войти админом (см. `Seed:Admin` в `appsettings.Development.json`:
   `admin@obratka.local` / `***REMOVED***`).
5. Проверка бота:
   - **Тест «жив»:** напишите боту `/wisdom` (в личке) или добавьте его в группу и напишите `/wisdom`
     (или «Обратка, мудрость» при выключенном privacy) — придёт философская мысль.
   - **Привязка:** кабинет → «Уведомления» → «Подключить Telegram» → Start → статус «привязан».
   - **Группа для алертов:** `/chatid` в группе → id в `AdminChatIds` → перезапуск Web API.

## 9. Диагностика
- Все исходящие запросы логируются в Seq: ищите `[telegram:send]` (type, chatId, ok/error, code).
- **Бот заблокирован пользователем** → в логе `канал недоступен (бот заблокирован) code=403`, мониторинг
  продолжает работать, повторных попыток нет.
- **Rate-limit** → `rate-limit code=429 ... без ретраев`.
- Сообщения не приходят, хотя «привязан» → проверьте доступ к `api.telegram.org` (§4) и что у мониторинга
  включён переключатель уведомлений (страница «Уведомления»).
- Бот не отвечает на `/start` → проверьте `Enabled=true`, корректность токена и сетевой доступ (long-poll
  тоже ходит на `api.telegram.org`).
